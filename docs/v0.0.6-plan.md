# SerialX v0.0.6 开发计划

## 版本定位

`v0.0.6` 定位为“主题与桌面体验增强版”。

这一版不再优先解决底层稳定性问题，而是围绕以下两个方向展开：

- 让应用在视觉层面更完整，具备浅色 / 深色模式和个性化壁纸能力
- 让工具页和工作区行为更统一、更顺手，减少风格割裂和连接后的额外操作

## 版本目标

### 1. 主题系统成型

- 支持浅色模式 / 深色模式切换
- 主题切换覆盖应用壳层、侧栏、终端周边控件、弹窗、下拉菜单等核心区域
- 主题状态持久化保存，重启后自动恢复

### 2. 桌面个性化增强

- 支持自定义壁纸
- 壁纸可以与主题协同工作，不影响终端主区域可读性
- 提供恢复默认和关闭壁纸的基础能力

### 3. 工具页风格统一

- 进制转换工具在布局、色彩、控件层级上统一到当前主应用风格
- 独立窗口模式下也具备完整的品牌感和一致性

### 4. 终端信息表达更清晰

- 控制台时间戳显示统一成 `[` `]` 包裹形式
- 时间戳颜色层级更明确，与正文和类型前缀区分开

### 5. 会话切换更顺手

- 连接新串口后，标签页自动切换到新串口
- 在多分屏场景下保持行为可预期，不打乱已有工作区结构

## 功能范围

## 一、主题与壁纸系统

### 计划项

- 建立全局主题状态：`dark` / `light`
- 梳理应用壳层 CSS 变量，减少当前颜色硬编码
- 将主题状态纳入设置持久化与会话恢复
- 支持自定义壁纸路径或资源引用
- 支持壁纸启用 / 禁用、恢复默认
- 为终端主区域保留足够对比度，避免壁纸影响正文可读性

### 验收标准

- 切换主题时主界面和弹窗颜色同步变化
- 重启应用后主题状态和壁纸配置仍然生效
- 壁纸启用后不会造成文本、按钮、菜单无法辨认

## 二、进制转换工具统一改造

### 计划项

- 重做转换器窗口布局，统一头部、输入区、结果区、按钮层级
- 与主应用共享主题变量和部分视觉语言
- 优化移动尺寸和窄窗口下的排版

### 验收标准

- 转换器独立窗口与主界面风格一致
- 布局在常见窗口尺寸下无明显拥挤和错位

## 三、终端时间戳显示优化

### 计划项

- 调整日志格式化逻辑，将时间戳显示为 `[12:34:56.789]`
- 时间戳使用独立颜色
- 保持 `TX / RX / INFO / ERR` 前缀层级清晰

### 验收标准

- 新日志时间戳格式统一
- 时间戳与正文在深色 / 浅色模式下都能清楚区分

## 四、连接后自动切换标签页

### 计划项

- 新串口连接成功后，自动激活对应标签页
- 多分屏场景下优先切到包含该串口的 pane
- 保持与侧栏选中状态同步

### 验收标准

- 连接新串口后无需手动找标签页
- 不会因为自动切换造成 pane 状态错乱

## 里程碑

### M1：主题基础

- 浅色 / 深色主题状态
- CSS 变量整理
- 基础持久化

### M2：壁纸与工具统一

- 自定义壁纸
- 进制转换工具重做

### M3：终端与会话细化

- 时间戳样式优化
- 新串口自动切换标签页
- 收口细节和回归验证

## 风险与注意事项

- 当前应用中仍有不少颜色直接写在组件里，主题化前需要先梳理变量边界
- 壁纸引入后容易影响可读性，需要为终端区域和表单区域做遮罩层处理
- 自动切换标签页要避免打断用户在多分屏中的当前操作

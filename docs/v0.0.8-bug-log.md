### BUG-001 Terminal Select All Was Not Working

- Status: `fixed`
- Date: 2026-03-24
- Reporter: user
- Symptoms: terminal full select could not be used reliably, including the dedicated full-select action.
- Root cause: the implementation used a browser `Range` on the terminal root element, but xterm renders terminal content through its own internal layers, so selecting the outer DOM node did not produce a reliable terminal-wide selection.
- Fix summary: switched full-select to xterm's native `selectAll()` path, updated selected text state after selection, and added `Ctrl/Cmd+A` handling to route through the same terminal-native full-select flow.
- Verification: `npm run build`
- Related files: `src/components/TerminalDisplay.vue`

### BUG-002 Appearance Settings Could Become Unopenable After Wallpaper Was Enabled

- Status: `fixed`
- Date: 2026-03-24
- Reporter: user
- Symptoms: after enabling wallpaper and using the app for a while, the appearance settings window could intermittently fail to open again.
- Root cause: the appearance settings route reused the main app shell and still performed wallpaper data-url loading even though the appearance window itself does not render the wallpaper layer, increasing unnecessary image loading and making the secondary renderer more fragile when large local wallpapers were enabled; the main-process reopen path also only focused an existing appearance window without explicitly restoring or recreating after renderer failure.
- Fix summary: skipped wallpaper data-url loading entirely in the appearance window route, ensured an existing appearance window is restored and shown before focusing, and added renderer-gone cleanup so a failed appearance window can be recreated cleanly on the next open.
- Verification: `npm run build`
- Related files: `src/App.vue`, `electron/main.js`

### BUG-003 Split Panes Could Not Be Reordered By Dragging

- Status: `fixed`
- Date: 2026-03-24
- Reporter: user
- Symptoms: after splitting the workspace into multiple panes, users could resize panes and drag tabs between panes, but could not drag an entire pane to swap its position with another pane.
- Root cause: the split workspace only implemented drag-and-drop for tabs, with no pane-level drag state, reorder logic, or dedicated drop feedback for pane groups.
- Fix summary: added pane-level drag state, a dedicated drag handle in each pane header, pane reorder logic that preserves pane width ordering, and visual drop feedback on pane groups without interfering with existing tab drag behavior.
- Verification: `npm run build`
- Related files: `src/components/TerminalView.vue`

### BUG-004 Appearance Window Stayed Open After Main Window Closed

- Status: `fixed`
- Date: 2026-03-24
- Reporter: user
- Symptoms: closing the main window could leave the appearance settings window open by itself, which broke the expected main-window-and-child-window lifecycle.
- Root cause: the appearance settings window was created as an independent `BrowserWindow` without being parented to the main window, and the main window close path did not explicitly close the appearance window first.
- Fix summary: made the appearance settings window a child of the main window and explicitly close it when the main window closes, so the auxiliary settings window now follows the main window lifecycle.
- Verification: `npm run build`
- Related files: `electron/main.js`

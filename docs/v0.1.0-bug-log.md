### BUG-001 Standalone Converter Window Missed App Theme Variables

- Status: `fixed`
- Date: 2026-03-25
- Reporter: user
- Symptoms: the standalone converter window inherited converter-specific CSS variables, but its close/pin controls and shared panel styles also depended on app-level theme variables such as `--app-danger-*` and `--app-workspace-*`, causing the standalone tool window to diverge from the main theme system.
- Root cause: the converter route does not render the main `app-container`, so it never received the root theme variable set that the main window applies.
- Fix summary: defined the required app-level semantic and workspace variables directly in `DataConverter.vue` so the standalone converter window now resolves the same theme colors without depending on the main window wrapper.
- Verification: `npm run build`
- Related files: `src/components/DataConverter.vue`

### BUG-002 Terminal Context Menu Position Was Incorrect

- Status: `fixed`
- Date: 2026-03-25
- Reporter: user
- Symptoms: the terminal right-click menu could appear in the wrong place, especially near edges or after container/layout changes, which made it feel detached from the actual click position.
- Root cause: the context menu was rendered inside the terminal component tree and relied on local fixed positioning, so it was still affected by the surrounding container hierarchy and viewport edge cases.
- Fix summary: moved the terminal context menu to a `Teleport`-based body layer, added viewport clamping after open, and closed the menu on scroll/resize so stale coordinates no longer persisted.
- Verification: `npm run build`
- Related files: `src/components/TerminalDisplay.vue`

### BUG-003 Split Panes Did Not Reflow When Sidebar Expanded Or Collapsed

- Status: `fixed`
- Date: 2026-03-25
- Reporter: user
- Symptoms: when the sidebar expanded, split panes did not shrink with the remaining workspace width; when the sidebar collapsed, split panes also did not grow back immediately.
- Root cause: pane width recalculation in `TerminalView` only listened to `window.resize`, but sidebar toggling changes the width of the workspace container itself without firing a window resize event.
- Fix summary: added a `ResizeObserver` on the split workspace content container so pane widths are recalculated whenever the available layout width changes, including sidebar expand/collapse transitions.
- Verification: `npm run build`
- Related files: `src/components/TerminalView.vue`

### BUG-004 Workspace Snapshot Restore Did Not Fully Refresh Split Panes

- Status: `fixed`
- Date: 2026-03-25
- Reporter: internal
- Symptoms: after applying a saved workspace snapshot, the store layout could already be updated, but `TerminalView` would continue rendering the previous local pane structure until some later interaction forced a resync.
- Root cause: `TerminalView` copied `workspaceLayout` only during initial setup and then pushed later local changes back into the store, but it did not watch the store layout itself for external updates such as snapshot restore.
- Fix summary: added a guarded workspace layout watcher in `TerminalView` so externally applied layout snapshots now rebuild local split panes and pane widths immediately without creating update loops.
- Verification: `npm run build`
- Related files: `src/components/TerminalView.vue`

### BUG-005 Restored Workspace Layout Could Keep Stale Or Duplicated Tabs

- Status: `fixed`
- Date: 2026-03-25
- Reporter: codex
- Symptoms: after restoring workspace state or applying snapshots, pane layouts could retain tabs for ports that were no longer connected, and duplicated tabs could survive across panes until a later port list change happened.
- Root cause: pane cleanup for disconnected tabs only ran inside the `connectedPorts` watcher, but initial restore and snapshot application can hydrate `workspaceLayout` before any watcher change occurs.
- Fix summary: moved connected-port filtering and duplicate-tab cleanup into the shared pane structure normalization path, and also reclaim empty panes there so restored workspace state is immediately normalized.
- Verification: `npm run build`
- Related files: `src/components/TerminalView.vue`

### BUG-006 Loop Send Interval Was Hard-Clamped To 100ms

- Status: `fixed`
- Date: 2026-03-25
- Reporter: user
- Symptoms: setting the loop-send interval below `100ms` had no effect, so values like `5ms` never actually ran at the requested cadence.
- Root cause: both the store scheduler and the loop interval input field enforced a hard minimum of `100ms`, so lower values were blocked in the UI and clamped again at runtime.
- Fix summary: removed the previous `100ms` hard clamp, lowered the loop-send minimum interval to `1ms` in both the UI and the scheduler, and normalized updates in the store so all write paths share the same lower bound.
- Verification: `npm run build`
- Related files: `src/stores/serial.js`, `src/components/SerialPanel.vue`

### BUG-007 Command Palette Sent Commands To Disconnected Selected Port

- Status: `fixed`
- Date: 2026-05-15
- Reporter: codex
- Symptoms: the command palette could dispatch a saved command when a port was selected but no longer connected, because Enter/click only required a selected port.
- Root cause: the palette send guard checked `selectedPort` presence instead of the active connection state returned by the serial store.
- Fix summary: added a selected-port connection computed guard, reused it for Enter/click send dispatch, and marked the palette target as inactive unless the selected port is connected.
- Verification: `npm run build`
- Related files: `src/components/CommandPalette.vue`


### BUG-008 Command Manager Accepted Whitespace-Only Commands

- Status: `fixed`
- Date: 2026-05-15
- Reporter: codex
- Symptoms: the common-command editor could save names, payloads, or groups that only contained whitespace, creating blank-looking command entries or invisible groups.
- Root cause: the command manager only checked raw truthiness and passed untrimmed fields to the serial store.
- Fix summary: normalized command names, payloads, and groups before saving or duplicating commands, and defaulted blank groups back to `默认`.
- Verification: `npm run build`
- Related files: `src/components/CommandsModal.vue`

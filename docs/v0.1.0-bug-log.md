### BUG-017 Clear-All Send History Warning Understated Draft Loss

- Status: `fixed`
- Date: 2026-05-16
- Reporter: scheduled follow-up
- Symptoms: the clear-all send-history confirmation only warned about deleting history, while the action also cleared the current send input draft for that port.
- Root cause: the confirmation and follow-up log described the history deletion but omitted the coupled draft reset performed after clearing history.
- Fix summary: updated the destructive confirmation to name the target port and explicitly state that the current send input draft will also be cleared; updated the resulting log message to match.
- Verification: `npm run build`
- Related files: `src/components/SerialPanel.vue`, `docs/v0.1.0-bug-log.md`

### BUG-016 Terminal Search Could Keep Stale Match Counts After Clearing Query

- Status: `fixed`
- Date: 2026-05-16
- Reporter: scheduled follow-up
- Symptoms: clearing or replacing the terminal search query could leave the visible search status/count on an older xterm search result until the addon emitted another result change.
- Root cause: search result callbacks were applied unconditionally, while cleared queries and failed/empty search runs did not invalidate older asynchronous result updates.
- Fix summary: added a monotonic search request token, centralized xterm search result application, reset match state when the query is empty or not found, and ignored result callbacks after the query has been cleared.
- Verification: `npm run build`
- Related files: `src/components/TerminalDisplay.vue`, `docs/v0.1.0-regression-checklist.md`

### BUG-015 Command Palette Empty State Lacked Actionable Guidance

- Status: `fixed`
- Date: 2026-05-15
- Reporter: scheduled follow-up
- Symptoms: when the command palette had no matches, no enabled commands, or no saved commands, the empty state only showed a generic message and did not clearly tell users what to do next.
- Root cause: the palette empty state did not distinguish searchable results from disabled/saved command states, and it had no dedicated hint for the manager workflow.
- Fix summary: added state-aware empty titles, details, and tip chips for search, disabled commands, and first-use states so users know whether to adjust search, enable commands, or add commands first.
- Verification: `npm run build`
- Related files: `src/components/CommandPalette.vue`, `docs/v0.1.0-regression-checklist.md`

### BUG-013 Command Palette Long Command Overflow

- Status: `fixed`
- Date: 2026-05-15
- Reporter: scheduled follow-up
- Symptoms: very long common command names or payloads could expand command palette rows past the modal width.
- Root cause: command palette item labels and payload chips did not define shrinkable widths or text overflow rules.
- Fix summary: added `min-width: 0` and ellipsis truncation to command names, constrained payload chips, and kept the existing tooltip title for full payload inspection.
- Verification: `npm run build`
- Related files: `src/components/CommandPalette.vue`, `docs/v0.1.0-regression-checklist.md`

### BUG-001 Standalone Converter Window Missed App Theme Variables

- Status: `fixed`
- Date: 2026-03-25
- Reporter: user
- Symptoms: the standalone converter window inherited converter-specific CSS variables, but its close/pin controls and shared panel styles also depended on app-level theme variables such as `--app-danger-*` and `--app-workspace-*`, causing the standalone tool window to diverge from the main theme system.
- Root cause: the converter route does not render the main `app-container`, so it never received the root theme variable set that the main window applies.
- Fix summary: defined the required app-level semantic and workspace variables directly in `DataConverter.vue` so the standalone converter window now resolves the same theme colors without depending on the main window wrapper.
- Verification: `npm run build`
- Related files: `src/components/DataConverter.vue`

### BUG-002 Terminal Context Menu Position Was Incorrect

- Status: `fixed`
- Date: 2026-03-25
- Reporter: user
- Symptoms: the terminal right-click menu could appear in the wrong place, especially near edges or after container/layout changes, which made it feel detached from the actual click position.
- Root cause: the context menu was rendered inside the terminal component tree and relied on local fixed positioning, so it was still affected by the surrounding container hierarchy and viewport edge cases.
- Fix summary: moved the terminal context menu to a `Teleport`-based body layer, added viewport clamping after open, and closed the menu on scroll/resize so stale coordinates no longer persisted.
- Verification: `npm run build`
- Related files: `src/components/TerminalDisplay.vue`

### BUG-003 Split Panes Did Not Reflow When Sidebar Expanded Or Collapsed

- Status: `fixed`
- Date: 2026-03-25
- Reporter: user
- Symptoms: when the sidebar expanded, split panes did not shrink with the remaining workspace width; when the sidebar collapsed, split panes also did not grow back immediately.
- Root cause: pane width recalculation in `TerminalView` only listened to `window.resize`, but sidebar toggling changes the width of the workspace container itself without firing a window resize event.
- Fix summary: added a `ResizeObserver` on the split workspace content container so pane widths are recalculated whenever the available layout width changes, including sidebar expand/collapse transitions.
- Verification: `npm run build`
- Related files: `src/components/TerminalView.vue`

### BUG-004 Workspace Snapshot Restore Did Not Fully Refresh Split Panes

- Status: `fixed`
- Date: 2026-03-25
- Reporter: internal
- Symptoms: after applying a saved workspace snapshot, the store layout could already be updated, but `TerminalView` would continue rendering the previous local pane structure until some later interaction forced a resync.
- Root cause: `TerminalView` copied `workspaceLayout` only during initial setup and then pushed later local changes back into the store, but it did not watch the store layout itself for external updates such as snapshot restore.
- Fix summary: added a guarded workspace layout watcher in `TerminalView` so externally applied layout snapshots now rebuild local split panes and pane widths immediately without creating update loops.
- Verification: `npm run build`
- Related files: `src/components/TerminalView.vue`

### BUG-005 Restored Workspace Layout Could Keep Stale Or Duplicated Tabs

- Status: `fixed`
- Date: 2026-03-25
- Reporter: codex
- Symptoms: after restoring workspace state or applying snapshots, pane layouts could retain tabs for ports that were no longer connected, and duplicated tabs could survive across panes until a later port list change happened.
- Root cause: pane cleanup for disconnected tabs only ran inside the `connectedPorts` watcher, but initial restore and snapshot application can hydrate `workspaceLayout` before any watcher change occurs.
- Fix summary: moved connected-port filtering and duplicate-tab cleanup into the shared pane structure normalization path, and also reclaim empty panes there so restored workspace state is immediately normalized.
- Verification: `npm run build`
- Related files: `src/components/TerminalView.vue`

### BUG-006 Loop Send Interval Was Hard-Clamped To 100ms

- Status: `fixed`
- Date: 2026-03-25
- Reporter: user
- Symptoms: setting the loop-send interval below `100ms` had no effect, so values like `5ms` never actually ran at the requested cadence.
- Root cause: both the store scheduler and the loop interval input field enforced a hard minimum of `100ms`, so lower values were blocked in the UI and clamped again at runtime.
- Fix summary: removed the previous `100ms` hard clamp, lowered the loop-send minimum interval to `1ms` in both the UI and the scheduler, and normalized updates in the store so all write paths share the same lower bound.
- Verification: `npm run build`
- Related files: `src/stores/serial.js`, `src/components/SerialPanel.vue`

### BUG-007 Command Palette Sent Commands To Disconnected Selected Port

- Status: `fixed`
- Date: 2026-05-15
- Reporter: codex
- Symptoms: the command palette could dispatch a saved command when a port was selected but no longer connected, because Enter/click only required a selected port.
- Root cause: the palette send guard checked `selectedPort` presence instead of the active connection state returned by the serial store.
- Fix summary: added a selected-port connection computed guard, reused it for Enter/click send dispatch, and marked the palette target as inactive unless the selected port is connected.
- Verification: `npm run build`
- Related files: `src/components/CommandPalette.vue`

### BUG-009 Send History Selector Disabled While Offline

- Reporter: codex
- Symptoms: recent send-history entries could not be selected to refill the send input after a port disconnected, even though history review and editing are safe offline actions.
- Root cause: the history selector inherited the same connection guard as actions that actually transmit data.

### BUG-010 Main Send Control Allowed Offline Dispatch Attempts

- Status: `fixed`
- Date: 2026-05-15
- Reporter: codex
- Symptoms: the main send input action could still be attempted while the selected port was disconnected, especially through Enter-send paths that bypassed the visual button state.
- Root cause: the primary send disabled state only checked empty input and HEX validity, and `handleSend` did not stop disconnected ports before calling the store send routine.
- Fix summary: added the active connection state to the main send disabled computed guard and added an explicit disconnected-port warning/log guard inside `handleSend`.
- Verification: `npm run build`
- Related files: `src/components/SerialPanel.vue`

### BUG-011 Send History Delete Left Stale Input

- Status: `fixed`
- Date: 2026-05-15
- Reporter: codex
- Symptoms: deleting the selected send-history entry could leave the deleted payload visible in the send input and the keyboard history draft/index out of sync.
- Root cause: delete-selected only removed the store entry and cleared the dropdown selection, without reconciling the input field or history navigation state.
- Fix summary: choose the next remaining history item before deletion, reset history navigation state, sync the send input/store value when it matched the removed payload, and log the resulting action.
- Verification: `npm run build`
- Related files: `src/components/SerialPanel.vue`

### BUG-012 Command Manager Allowed Duplicate Saved Commands

- Status: `fixed`
- Date: 2026-05-16
- Reporter: codex
- Symptoms: the common-command manager could save multiple commands with the same trimmed name, payload, and group, making command lists look duplicated and leaving repeated duplicate actions indistinguishable.
- Root cause: command validation only rejected blank normalized names or payloads and did not compare normalized command triples against existing entries; duplicate action always used the same `副本` suffix.
- Fix summary: added case/trim-insensitive duplicate detection for name, payload, and normalized group while excluding the current command during edits, and made duplicate action increment copy names until the new entry is unique.
- Verification: `npm run build`
- Related files: `src/components/CommandsModal.vue`

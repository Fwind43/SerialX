# SerialX v0.1.0 Comprehensive ToDo

This checklist is the working backlog for the `v0.1.0` cycle. It turns the version plan into concrete, reviewable items so scheduled iterations can pick one small, low-risk task at a time.

## Guiding Principles

- Keep SerialX a local-first desktop serial debugging tool.
- Do not add cloud sync, account systems, remote storage, or online-dependent core features.
- Prefer small, verifiable improvements over large rewrites.
- Treat UI behavior changes as product changes: update docs and verify the related workflow.
- Record confirmed bug fixes in `docs/v0.1.0-bug-log.md`.
- Keep README, plan, regression checklist, and shipped behavior aligned.

## Status Legend

- `[x]` done in the current `v0.1.0` cycle
- `[ ]` open / candidate for scheduled iterations
- `[~]` partially done, needs more verification or polish
- `[>]` deferred unless priority changes

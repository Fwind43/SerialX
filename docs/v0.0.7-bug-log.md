### BUG-001 Terminal Scrollbar Was Not Usable

- Status: `fixed`
- Date: 2026-03-24
- Reporter: user
- Symptoms: the scrollbar inside the terminal area could not be dragged or used normally.
- Root cause: custom xterm layout overrides caused the terminal content layers to occupy the scrollbar gutter area, so clicking near the scrollbar hit the terminal content layer instead of the scrollbar itself.
- Fix summary: restored viewport interaction, normalized the viewport width, and reserved an explicit scrollbar gutter by shrinking the internal xterm screen and row layers away from the right-side scrollbar area.
- Verification: `npm run build`
- Related files: `src/components/TerminalDisplay.vue`

### BUG-002 Custom Wallpaper Could Not Load Local Image Files

- Status: `fixed`
- Date: 2026-03-24
- Reporter: user
- Symptoms: selecting a local wallpaper failed with `Not allowed to load local resource: file:///...`, and after the loading path was fixed the wallpaper was still visually hidden behind opaque sidebar and main-area surfaces.
- Root cause: the renderer directly used `file:///` URLs for desktop images, which the current Electron window security settings reject, and the app shell surfaces in both light and dark themes were too opaque to let the wallpaper layer show through.
- Fix summary: added an Electron IPC path to read the wallpaper file in the main process and return a `data:` URL, switched the renderer to use a dedicated wallpaper layer, reduced shell panel opacity when wallpaper is enabled so the sidebar and main workspace can reveal the wallpaper while staying readable, and added separate opacity controls for sidebar, workspace, and terminal surfaces so users can tune each area independently.
- Verification: `npm run build`
- Related files: `electron/main.js`, `electron/preload.js`, `src/App.vue`, `src/stores/serial.js`, `src/components/SerialSidebar.vue`, `src/components/TerminalView.vue`, `src/components/TerminalDisplay.vue`

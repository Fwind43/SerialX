# SerialX v0.0.5 开发计划

## 版本定位

`v0.0.5` 定位为“稳定性 + 效率优化”版本。

在 `v0.0.4` 已经完成一轮 UI 收敛之后，这一版不再以大幅视觉改造为重点，而是聚焦三件事：

- 提升多串口、多分屏场景下的稳定性
- 优化终端、搜索、发送等高频操作体验
- 整理设置与会话管理，为后续版本扩展留出结构空间

## 版本目标

### 1. 稳定可靠

- 串口连接、断开、异常退出后的状态恢复更一致
- 分屏关闭、切换、自动回收时不再出现残留宽度或状态错位
- 端口刷新后尽量保留用户的当前工作上下文

### 2. 操作高效

- 搜索、复制、发送、切换会话等高频动作更顺手
- 用户在常见调试路径上减少多余点击
- 错误提示更直接，不依赖开发者控制台

### 3. 结构可持续

- 设置项和会话状态管理继续收口
- 导入导出、布局恢复等能力具备更清晰的数据边界
- 为日志导出、会话保存等后续功能预留基础结构

## 开发范围

## 一、连接与会话稳定性

### 目标

让串口状态、侧栏选择、标签页、分屏布局在各种连接变化下保持一致。

### 计划项

- 优化串口异常断开后的 UI 回收逻辑
- 优化端口列表刷新后的选中状态保留
- 完善连接失败、端口占用、发送失败等错误提示
- 继续加强分屏宽度与 pane 生命周期同步
- 关闭最后一个已断开的 pane 时自动回收布局

### 验收标准

- 已连接串口断开后，不出现空白 pane 占位
- 侧栏、标签、当前面板三者状态保持一致
- 端口刷新后，未移除的当前端口应尽量保持选中
- 常见连接失败场景有明确界面提示

## 二、终端体验优化

### 目标

让终端继续作为主区域，同时提升搜索、复制、查找定位的效率。

### 计划项

- 优化搜索浮层交互，支持更顺手的键盘行为
- 支持 `Enter` / `Shift+Enter` 在匹配项之间切换
- 继续整理搜索高亮方案，确保命中项可读性稳定
- 优化终端右键菜单，统一复制、清空、查找入口
- 评估大量日志场景下的渲染与搜索响应表现

### 验收标准

- 第一行匹配项不被搜索浮层遮挡
- 当前匹配项有清晰可辨的高亮
- 终端中选中文本后复制逻辑稳定
- 高量日志场景下搜索无明显卡顿或错误

## 三、发送工作流优化

### 目标

减少发送区的操作成本，让循环发送和常用命令更适合连续调试。

### 计划项

- 循环发送增加启动延时配置
- 循环发送增加失败次数阈值与自动停止能力
- 增加发送完成、发送失败的明确反馈
- 为发送输入补充快捷键行为
- 优化常用命令区的排序、置顶或分组方案

### 验收标准

- 循环发送的开始、暂停、继续、停止状态可预期
- 发送失败时用户能直接在界面中看到原因或结果
- 常用命令调用路径比当前更短
- 高频发送场景下无需频繁鼠标点按

## 四、设置与数据管理

### 目标

让设置系统从“能保存”提升到“更清晰、更安全、更可扩展”。

### 计划项

- 继续梳理设置结构，按基础、终端、高级分层
- 优化设置导入导出的错误校验和提示
- 为工作区布局恢复建立更明确的状态结构
- 为后续日志导出、会话快照保留数据接口

### 验收标准

- 设置项层次更清楚，低频项不过度打扰主界面
- 导入无效配置时有明确提示，不会静默失败
- 重启后可恢复关键工作区状态
- 数据结构不会和 UI 状态强耦合

## 版本里程碑

### M1：稳定性收口

重点内容：

- 连接、断开、异常提示
- 分屏宽度与 pane 生命周期同步
- 侧栏、标签、当前会话状态统一

目标结果：

- 先把最容易打断工作流的问题消掉

### M2：高频操作提效

重点内容：

- 搜索快捷行为
- 终端右键菜单
- 发送快捷键
- 循环发送增强

目标结果：

- 让高频动作更短、更顺手

### M3：设置与会话整理

重点内容：

- 设置分层
- 导入导出校验
- 布局恢复
- 数据结构整理

目标结果：

- 为后续功能迭代打基础

## 暂不纳入 v0.0.5 的内容

- 大规模视觉重设计
- 完整日志分析系统
- 云同步或远程配置同步
- 复杂脚本编排能力

这些方向可以在后续版本中单独规划，避免 `v0.0.5` 范围失控。

## 建议开发顺序

1. 先完成连接稳定性和分屏状态问题
2. 再优化搜索、复制、发送等高频体验
3. 最后整理设置结构和会话恢复能力

## 风险与注意事项

- 终端显示、搜索高亮、复制行为都和 `xterm` 行为强相关，改动时需要反复验证
- 多串口与多分屏状态是这一版最容易产生边界问题的区域
- 设置导入导出和布局恢复需要注意旧版本兼容性

## 文档维护方式

本文件作为 `v0.0.5` 的主计划文档使用。

开发过程中建议按以下方式维护：

- 新增需求时补充到对应模块
- 已完成项在开发记录或提交信息中关联
- 范围发生变化时优先更新本计划，再进入实现

# SerialX Local Data Boundaries

This document records the current local persistence and import/export boundary for the `v0.1.0` cycle. It is intentionally descriptive: it documents shipped behavior and does not add any remote storage requirement.

## Storage model

SerialX is a local-first desktop tool. Core serial workflows should not depend on network availability.

Persisted app data is stored on the local machine through the Electron settings bridge and browser local storage used by the renderer session.

## Data persisted locally

The current local state includes:

- theme and appearance settings, including the selected app theme, terminal appearance, custom theme presets, wallpaper-related appearance choices, and sidebar / workspace / terminal opacity settings
- serial defaults and control settings, including default connection options and send / display preferences
- auto-log settings
- common commands and command groups
- workspace layout, selected port state, saved workspace snapshots, and active snapshot metadata
- UI state used to restore the main workspace, such as sidebar and active panel state
- per-port send history stored in the renderer session so recent commands can be recalled locally

## Settings import/export boundary

Settings import/export is intended for user-controlled local backup or transfer of configuration-like state.

The exported settings payload currently covers:

- default serial settings
- auto-log settings
- display and control settings
- common commands
- app theme and terminal appearance
- custom appearance presets
- saved workspace snapshots
- workspace layout
- selected port
- app UI state

Importing settings restores these configuration and workspace fields into the local app state.

## Data outside settings import/export

The following data is intentionally outside the settings import/export payload unless a future iteration explicitly changes that boundary:

- raw terminal RX / TX logs
- ad-hoc exported log files created from terminal or all-port log export actions
- per-port send history kept in renderer local storage
- operating-system level serial driver state or physical device configuration
- remote, account, cloud, or collaboration data, which are deferred / non-goals for `v0.1.0`

## Regression expectations

For `v0.1.0`, persistence verification should confirm that restart and settings import/export preserve configuration and workspace state without requiring network access, while logs remain export-driven user files rather than hidden synchronized data.

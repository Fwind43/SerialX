# SerialX v0.1.0 Regression Checklist

This checklist defines the minimum manual verification baseline for `v0.1.0`.

## 1. Build

- `npm run build` passes
- app can launch successfully after build output is produced

## 2. Serial Basics

- port list can refresh normally
- a port can connect successfully
- a connected port can disconnect successfully
- connecting a new port creates or activates a usable terminal tab

## 3. Terminal Basics

- terminal receives data and displays TX / RX logs
- clear action works
- select and copy work
- full select works
- copy current screen works
- copy all logs works
- search opens, finds matches, and can navigate next / previous
- search status clearly shows empty, matched, and no-match states
- context menu actions work without breaking focus
- context menu appears inside the visible area and does not drift on resize or scroll

## 4. HEX Mode

- HEX receive mode displays bytes correctly
- HEX + ASCII mode remains aligned
- HEX continuation alignment does not break layout
- copied or exported content matches the displayed intent

## 5. Workspace

- split pane action creates a second pane
- pane resize works
- pane reorder works
- tab drag between panes works
- tab right-click actions work
- split panes reflow when the sidebar is expanded or collapsed
- pane cleanup after disconnect does not leave invalid active state
- selected port stays synchronized with active pane/tab
- saved workspace snapshots can be created, renamed, applied, and deleted
- the active workspace snapshot is visibly highlighted
- applying a saved workspace snapshot refreshes the local pane layout immediately

## 6. Commands and Send Flow

- plain text send works
- HEX send works
- loop send can start and stop
- send history can be recalled without widening the send area
- `ArrowUp` / `ArrowDown` can browse recent sends without losing the current draft
- common commands are grouped in the manager and command dropdown
- command manager search works by name, content, and group
- command duplication works in the command manager
- command selection and quick send remain usable

## 7. Appearance and Theme

- built-in light theme applies consistently
- built-in dark theme applies consistently
- appearance settings window opens normally
- changing theme in appearance settings updates the main window
- wallpaper enable/disable works
- sidebar / workspace / terminal transparency changes take effect
- standalone converter window keeps theme variables consistent

## 8. Auxiliary Windows

- appearance settings window follows main-window lifecycle
- converter window opens normally
- converter window close works
- converter always-on-top toggle works

## 9. Persistence

- restart preserves expected theme state
- restart preserves expected workspace layout
- restart preserves saved workspace snapshots and the active snapshot selection
- custom appearance settings restore correctly
- settings import/export does not break startup
- exporting all-port logs from the main menu works in both JSON and TXT

## 10. Release Gate

`v0.1.0` should not be tagged until:

- all high-frequency workflows above pass
- newly fixed bugs are recorded in `docs/v0.1.0-bug-log.md`
- release notes match actual shipped behavior
